#### 1. 核心SQL语句总结

**（1）修改表结构（ALTER TABLE）**
```sql
-- 添加列
ALTER TABLE 表名 
ADD COLUMN 列名 数据类型 [约束];

-- 示例（你今天敲过类似）
ALTER TABLE xs 
ADD COLUMN 奖学金等级 TINYINT NULL;

-- 删除列
ALTER TABLE 表名 
DROP COLUMN 列名;

-- 示例
ALTER TABLE xs 
DROP COLUMN 姓名;
```

**（2）修改表名（RENAME）**
```sql
RENAME TABLE 旧表名 TO 新表名;
-- 或
ALTER TABLE 旧表名 RENAME TO 新表名;

-- 你今天成功执行
USE xscj;
RENAME TABLE XS TO student;
```

**（3）表复制（重点！）**
```sql
-- 只复制表结构（推荐用于备份）
CREATE TABLE 新表名 LIKE 旧表名;

-- 复制表结构 + 所有数据
CREATE TABLE 新表名 AS 
SELECT * FROM 旧表名;

-- 你今天操作示例
CREATE TABLE kc_copy1 LIKE kc;                    -- 只复制结构
CREATE TABLE cj_copy2 AS SELECT * FROM xs_kc;     -- 复制结构+数据
```

**（4）删除表（慎用，不可恢复）**
```sql
DROP TABLE 表名;

-- 你今天执行
DROP TABLE kc_copy1;
```

**（5）查看表信息**
```sql
SHOW TABLES;                    -- 显示当前数据库所有表
DESC 表名;                      -- 显示表结构（推荐）
DESCRIBE 表名;                  -- 同上
SHOW COLUMNS FROM 表名 LIKE '列名';   -- 精确查看某一列
```

#### 3. 易错点提醒（以后避免）
- 创建表时最后一列后面不要多余逗号（MySQL有时会报错）
- `TINYINT(1)` 可以简写为 `TINYINT`
- 删除列/表操作不可恢复，操作前建议先 `SELECT * FROM 表名 LIMIT 5;` 确认数据
- 临时表命名建议统一加 `_copy` 或日期后缀，便于管理
