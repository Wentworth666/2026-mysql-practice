#### 2.2.3 数据库高级操作（开头部分，老师可能快速过了一遍）

- 查看数据库创建语句（超级实用，复制字符集用）：
    
    SQL
    
    ```
    SHOW CREATE DATABASE 库名;
    ```
    
    示例输出： CREATE DATABASE shop_db /*!40100 DEFAULT CHARACTER SET utf8mb4 */
- 修改数据库字符集（创建时忘了加utf8mb4可以用这个补）：
    
    SQL
    
    ```
    ALTER DATABASE 库名 DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
    ```
    
- 安全删除数据库（老师提醒过）：
    
    SQL
    
    ```
    DROP DATABASE IF EXISTS 库名;
    ```

#### 3.2.1.1 数据类型（今天重头戏，老师重点讲的全部内容）

**1. 数值类型（老师强调精度问题）**

- INT / INTEGER：普通整数（-2^31 ~ 2^31-1）
- BIGINT：超大整数（ID、手机号常用）
- FLOAT / DOUBLE：浮点数（不精确，小数运算有误差）
- DECIMAL(M,D)：精确小数（**钱、成绩、分数必须用这个**）
    - M：总位数（包括小数点前后）
    - D：小数位数
    - 示例：price DECIMAL(10,2) → 最多99999999.99

**2. 字符串类型（最常用，老师反复提）**

- CHAR(n)：固定长度（自动补空格，适合性别、状态码）
    - 示例：gender CHAR(1) → '男'、'女'
- VARCHAR(n)：可变长度（**最推荐！节省空间**）
    - 示例：name VARCHAR(50)、email VARCHAR(100)
- TEXT / LONGTEXT：超长文本（文章、备注、JSON）

**3. 日期/时间类型（老师说以后插入数据会用到）**

- DATE：年-月-日（2026-03-17）
- TIME：时:分:秒
- DATETIME：年-月-日 时:分:秒（最常用）
- TIMESTAMP：时间戳（自动更新，适合 create_time/update_time）

**4. 其他类型（老师可能简单提了）**

- ENUM('男','女')：枚举，只能选这些值
- SET：多选（不常用）

**老师课堂强调 + 易错点**

- DECIMAL 必须写 (M,D)，不写默认 DECIMAL(10,0)
- VARCHAR 必须写最大长度（如50），不写报错
- FLOAT/DOUBLE 不要用于钱（精度丢失）
- 所有字符串类型都推荐 utf8mb4 字符集（支持表情）
- Navicat 选类型时，VARCHAR 点“长度”填数字

**记忆口诀**（超好记～）

- 整数：INT 小 BIGINT 大
- 小数：DECIMAL 钱最准，FLOAT DOUBLE 爱出错
- 字符串：CHAR 定长补空格，VARCHAR 变长最省事
- 时间：DATE 只年月日，DATETIME 带时分秒最全
